# 위키 탐색·참고 (wiki-reference)

위키에서 **빠르게 정확히 찾아 출처와 함께** 답하고, 더 알고 싶은 사람에게 **공식 가이드 검색어**를 안내하는 스킬.
주 소비자는 **AI**다(사람이 직접 읽기보다 AI가 참고해 답함).

> 핵심 한 줄: **INDEX 먼저 → 그 파일만 읽기 → 출처 한 줄 → 위키에 없으면 솔직히 + 공식 검색어 안내.**

---

## 0. 위키가 어디 있나

- **`wiki/INDEX.md`** — ★ 항상 먼저 읽는 지도(198문서 3단 트리 + 한 줄 설명). 전체 로드 금지.
- **`wiki/<대분류>/<NN-제목>.md`** — 개별 문서 본문(요약 0, 통째 보존). 관련될 때만 골라 읽는다.
- **`reference/TaskWiki.md`** — 31k 원본. 평소엔 안 읽고 **`grep` 검색용**.
- **`gitwiki/`** — 같은 내용의 GitHub 위키 배포본(`Home.md`/`_Sidebar.md` + 플랫 페이지). 깃으로 호스팅하는 버전. 로컬 작업은 `wiki/`를 본다.

---

## 1. 조회 흐름 (on-demand)

1. **`wiki/INDEX.md`부터** 본다. 키워드와 한 줄 설명으로 후보 1~3개를 고른다.
2. INDEX로 애매하면 원본을 grep해 위치부터 짚는다:
   `grep -n "키워드" reference/TaskWiki.md` → 어느 대분류/문서인지 파악 후 해당 `wiki/...` 파일로.
3. **고른 그 파일만** 읽고 답한다. 본문 디테일을 요약하지 말고 필요한 부분을 그대로 인용/설명.
4. 답 끝에 **출처 한 줄**: `근거: wiki/02-지형과이동/03-발판 만들기.md`
5. 여러 문서에 걸치면 2~3개까지만 열고, 더 필요하면 사용자에게 범위를 좁혀 묻는다(한 번에 조금씩).

---

## 2. ★ "더 자세히 알고 싶다" → 공식 가이드 검색 안내

**이 위키의 각 문서 제목 = 메이플스토리 월드 공식 가이드 문서 제목과 동일하다.** 이걸 활용한다.

- 위키 본문으로 충분치 않거나, 사용자가 **더 깊이/최신/공식 원문**을 원하면:
  → **"공식 가이드에서 「<문서 제목>」 으로 검색하세요"** 라고 안내한다.
  → 구글 검색어 형태도 같이: **`메이플스토리 월드 <문서 제목>`**
- 예시 안내 문구:
  > 이 위키엔 여기까지 있어. 더 자세하거나 최신 내용은 메이플스토리 월드 공식 가이드에서 **「발판 만들기」** 로 검색하면 원문을 볼 수 있어. (구글: `메이플스토리 월드 발판 만들기`)
- 사용자가 원하면 WebSearch/WebFetch로 직접 공식 원문을 찾아줘도 된다. 단, **가져온 내용은 "공식 사이트 근거"라고 출처를 밝히고**, 위키 본문과 구분한다.

---

## 3. 정확성·출처 규칙 (엄수)

- 위키에 있으면 **파일 경로**를, 웹에서 가져왔으면 **URL/출처**를, 둘 다 없으면 **"내 추론"**이라고 명시.
- **확실/불확실을 구분**해서 말한다. 검증 안 된 단정 금지.
- 위키에 없는 주제면 숨기지 말고 "위키엔 없음" → §2로 공식 검색어 안내.

## 4. 빠른 grep 레시피

```bash
# 제목/키워드로 어느 문서인지
grep -rl "Foothold" wiki/ --include=*.md            # (예시) 키워드 든 파일 목록
grep -rn "FootholdComponent" wiki/                  # 줄까지
grep -n "텔레포트" reference/TaskWiki.md             # 원본에서 위치만 싸게
# 대분류 안에서만
grep -rn "사다리" wiki/02-지형과이동/
```
