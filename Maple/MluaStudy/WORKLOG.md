## 2. 위키 분할을 어떻게 했나 (★ HANDOFF 가정과 달랐던 핵심)

**HANDOFF §4는 "위키가 `#문서명` 마크다운 헤더로 잘려있다"고 가정했지만, 실제 원본엔 헤더가 없었다**(순수 텍스트 덤프). 그래서 다른 신뢰 신호로 잘랐다:

1. **문서 경계 = `^custom custom$` 마커.** 원본에 정확히 198개 → 문서 198개. (각 문서: `…custom custom / (빈줄) / 학습 과정 소개 / 본문…` 패턴.)
2. **대분류 경계 = 13개 대분류 이름이 본문에 글루된 줄.** (예: line 8059 `…확인합니다.지형과 이동`.) 단조 증가하는 마커 줄번호로 폴더를 나눔.
3. **제목이 앞 문서 끝에 붙어(글루) 있었다.** 예: `…불러옵니다.WorldConfig`, 심지어 한글+한글(`캐릭터 초기 속성 편집게임 섬네일 변경하기`)이라 정규식으로 못 자름. → **198개 제목을 사람이(=내가) 직접 읽어 권위 리스트로 스크립트에 하드코딩**했다.
4. 분할 스크립트: `tools/split_wiki.py` (repo에 보존됨). 198개 권위 제목 리스트가 이 안에 박혀있음. 재분할하려면 `python3 tools/split_wiki.py` (단, 그러면 wiki/를 덮어씀). 본문은 컨텍스트에 안 올리고 줄번호·`grep`/`python`으로만 처리함.

**파일 포맷**: 각 문서 = `# 제목` + `<!-- 출처 주석 -->` + 본문 통째. 파일명 `wiki/<대분류>/<순번-제목>.md`.
**INDEX 한 줄 설명**: 각 문서 `학습 과정 소개` 다음 문단에서 추출.

### 무손실 검증 결과 (중요)
- 원본 본문 줄과 분할본을 멀티셋 비교 → **진짜 본문 손실 0줄.**
- "사라진" 것처럼 보이는 줄들은 전부 (a) 글루된 제목줄(→ `#` 헤더로 이동) (b) 섹션 시작의 중복 네비 메뉴(→ INDEX.md가 대체) (c) `custom custom` 마커뿐. **디테일 손실 없음.**
